Weather Dashboard

Build a clean and simple weather app that fetches live data from a third-party API based on a user's city search.

üöÄ The Project

This is a perfect project to understand how to work with Third-Party APIs. We will build a simple dashboard where a user can search for any city name, and our app will call an external API (like OpenWeatherMap) to fetch and display the current temperature, humidity, and weather conditions.

With this project, you will master asynchronous JavaScript (Async/Await) and handling API data.

‚ú® Key Features

  • A search bar for the user to search for a city name.
  • Display live data fetched from the API (Temperature, Weather, Humidity, Wind Speed).
  • Loading state (while data is being fetched).
  • Error handling (if the city is not found or the API fails).

üõ†Ô∏è Step-by-Step Guide

Part 1: Get Your API Key

  1. First, go to OpenWeatherMap and create a free account.
  2. Copy your unique "API Key" from your account dashboard. This key is necessary to authorize your API requests.

Part 2: React UI Setup

  1. Create a new React app.
  2. Create a `Weather` component.
  3. Use `useState` to create state variables: `city` (for the search input), `weatherData` (to store the API response), `loading`, and `error`.
  4. Create a search bar (an input field and a button).

Part 3: API Fetching

  1. Create an `async` function, for example, `fetchWeather`.
  2. Inside this function, use a `try...catch` block for error handling.
  3. In the `try` block:
    • Set `setLoading(true)`.
    • Use the `fetch` function (or Axios) to call the OpenWeatherMap API. Pass your API key and the `city` state variable in the URL.
    • Wait (`await`) for the response and convert it to `.json()`.
    • If the response is `404` (Not Found), set an error.
    • Save the successful response to the state using `setWeatherData`.
  4. In the `catch` block, set the `error`.
  5. In the `finally` block, set `setLoading(false)`.
  6. Call this `fetchWeather` function on the search button's `onClick`.

Part 4: Displaying the Data

  1. Use conditional rendering.
  2. If `loading` is true, show "Loading...".
  3. If `error` exists, show the error message.
  4. If `weatherData` is available, create a new component (like `WeatherCard`), pass the data (temperature, icon, etc.) as props, and display it.

üî• Next Steps & Challenges

After completing the project, try these ideas:

  • **5-Day Forecast:** Along with the current weather, display the forecast for the next 5 days (OpenWeatherMap provides this data too).
  • **Geolocation:** Show the weather for the user's current location (using the browser's Geolocation API).
  • **Save Last Search:** Save the user's last searched city in `localStorage` so it becomes the default location when the app opens.
