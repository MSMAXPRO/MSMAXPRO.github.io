Chapter 1: Learn the Language: Kotlin

What is Kotlin?

Kotlin ek modern, statically-typed programming language hai jise JetBrains (Android Studio banane waali company) ne banaya hai. 2017 mein, Google ne ise Android development ke liye **first-class, official language** ghoshit kar diya. Yeh Java Virtual Machine (JVM) par chalti hai aur ise Java ka ek behtar, naya version maana jaata hai.

Agar aap Android app banana chahte hain, toh Kotlin seekhna zaroori hai. Yeh aapke code ko chhota (concise), safe, aur padhne mein aasaan banati hai.

Why Kotlin over Java?

Ho sakta hai aapne Java ka naam suna ho. Java 20 saal tak Android ki main language thi, lekin ab Google Kotlin ko prefer karta hai. Kyun?

  • Concise (Chhota Code): Jo kaam Java mein 40 line mein hota hai, Kotlin use 10 line mein kar sakti hai. Kam code ka matlab hai kam bugs.
  • Null Safety (Sabse Important): Kotlin ka sabse bada feature. Yeh `NullPointerException` (app crash) ko jad se khatam karne mein help karta hai. Hum ispar detail mein baat karenge.
  • 100% Interoperable with Java: Aap apne Kotlin project mein Java code aur Java project mein Kotlin code, dono ek saath chala sakte hain.
  • Modern Features: Ismein data classes, extension functions, aur coroutines jaise modern features hain jo development ko fast banate hain.

Environment Setup

Kotlin seekhne ke liye aapko 2 cheezein chahiye:

  1. Android Studio: Yeh official IDE (Integrated Development Environment) hai Android apps banane ke liye. Ismein Kotlin pehle se installed aata hai. Hum ise agle chapter mein detail mein setup karenge.
  2. Kotlin Playground: Abhi ke liye, aapko kuch bhi install karne ki zaroorat nahi hai. Aap seedha Kotlin Playground par jaakar online hi saare code examples try kar sakte hain.

Kotlin Basics: Building Blocks

The "Hello, World!" Program

Har language ki shuruaat "Hello, World!" se hoti hai. Yeh Kotlin ka basic structure hai.

// Yeh ek single-line comment hai
fun main() {
    println("Hello, CodeWithMSMAXPRO!")
}
  • fun: Yeh "function" ka keyword hai. Function ek code ka block hota hai jise hum koi naam dete hain.
  • main(): Yeh ek special function hai. Jab bhi aap koi Kotlin program run karte hain, code yahaan se shuru hota hai.
  • println(...): Yeh ek built-in function hai jo text ko console par print karta hai aur ek nayi line add kar deta hai.
  • "Hello, ...": Yeh ek 'String' (text) hai. String hamesha double quotes (`" "`) ke andar likhi jaati hai.

Variables: `val` vs `var`

Variables data ko store karne ke liye containers hote hain. Kotlin mein 2 tarah ke variables hote hain, aur yeh bahut important hai.

`val` (Value) - Read-Only

val ka istemaal un variables ke liye hota hai jinki value **change nahi ho sakti** (immutable). Ise aap ek "constant" (jaise const in JavaScript) samajh sakte hain. Hamesha `val` use karne ki koshish karein jab tak zaroori na ho.

val name = "MSMAXPRO"  // Ek String variable banaya
println(name)

// name = "Aman"  // <-- YEH ERROR DEGA! (Val cannot be reassigned)

`var` (Variable) - Mutable

var ka istemaal un variables ke liye hota hai jinki value ko aap **baad mein change kar sakte hain** (mutable). Ise aap (let in JavaScript) samajh sakte hain.

var score = 0          // Ek Integer (number) variable banaya
println(score)  // Output: 0

score = 100         // Ise change karna allowed hai
println(score)  // Output: 100

Basic Data Types & Type Inference

Kotlin statically-typed hai, yaani har variable ka ek type (jaise Number ya Text) hota hai. Lekin Kotlin smart hai, yeh "Type Inference" ka use karta hai. Iska matlab hai ki aapko type batane ki zaroorat nahi hai, Kotlin khud samajh jaata hai.

val myString = "This is a String"   // Kotlin samajh gaya yeh String hai
val myInt = 25                     // Yeh ek Int (Integer) hai
val myDouble = 3.14                 // Yeh ek Double (decimal number) hai
val myBoolean = true               // Yeh ek Boolean (true/false) hai

Aap chahein toh type khud bhi bata sakte hain (explicitly), aise:

val age: Int = 20
val pi: Double = 3.14
val username: String = "msmaxpro"

String Templates

Yeh text ke andar variables daalne ka aasaan tareeka hai.

val name = "Aman"
val score = 100

// Variable use karne ke liye $ ka istemaal karein
val greeting = "Hello, $name! Welcome."
println(greeting) // Output: Hello, Aman! Welcome.

// Calculation ke liye ${...} ka istemaal karein
val message = "Your score is ${score + 50}"
println(message)  // Output: Your score is 150

The Most Important Feature: Null Safety

Java (aur kai puraani languages) mein sabse bada problem `NullPointerException` (NPE) hai. Yeh tab hota hai jab aap kisi aise variable par kaam karne ki koshish karte hain jiski value `null` (khaali) hai. Isse aapki app **CRASH** ho jaati hai.

Kotlin is problem ko type system mein hi solve kar deta hai.

Non-Nullable Types (Default)

Kotlin mein, by default, koi bhi variable `null` nahi ho sakta.

var name: String = "MSMAXPRO"
// name = null  // <-- YEH ERROR DEGA! (Compilation Error)
// Isse aapki app crash hone se bach gayi.

Nullable Types (?)

Agar aap chahte hain ki ek variable `null` ho sake, toh aapko Kotlin ko explicitly batana padega. Yeh `?` (question mark) laga kar kiya jaata hai.

var name: String? = "MSMAXPRO"
name = null // Yeh ab allowed hai
println(name) // Output: null

Handling Nullable Types

Ab jab `name` null ho sakta hai, toh aap `name.length` call nahi kar sakte, kyunki `null` ki koi length nahi hoti. Kotlin aapko aisa karne se rokega. Ise handle karne ke 3 tareeke hain:

1. Safe Call Operator: `?.` (Best Method)

Yeh kehta hai: "Agar variable null *nahi* hai, toh property access karo. Agar woh null hai, toh kuch mat karo (sirf `null` return karo)."

var name: String? = null
val length = name?.length
println(length) // Output: null (App crash nahi hui!)

2. The Elvis Operator: `?:` (Default Value)

Yeh Safe Call ke saath use hota hai. Yeh kehta hai: "Agar left waali cheez `null` hai, toh right waali default value use karo."

var name: String? = null
val length = name?.length ?: 0

println(length) // Output: 0 (Kyunki name null tha, default value 0 use hui)

3. Non-Null Asserted Call: `!!` (Dangerous)

Yeh operator (`!!`) Kotlin ko zabardasti kehta hai: "Main 100% sure hoon ki yeh variable null nahi hai, code ko run karo." Agar aap galat hue aur variable null nikla, toh aapki **app crash ho jaayegi** (`NullPointerException`). Ise avoid karein.

var name: String? = null
val length = name!!.length // <-- APP CRASH HOGI!

Functions in Kotlin

Functions reusable code blocks hote hain. Humne `main` function pehle hi dekha hai. Ab dekhte hain ki apne functions kaise banayein.

Defining and Calling Functions

Functions `fun` keyword se shuru hote hain. `()` ke andar parameters (inputs) define kiye jaate hain.

// 1. Ek simple function define karna
fun greet() {
    println("Hello from my function!")
}

// 2. Function ko call (run) karna
greet() // Output: Hello from my function!

Parameters and Return Types

Aap functions ko data (parameters) de sakte hain, aur unse data (return value) waapis le sakte hain.

// Parameters: name (String type ka)
fun greetUser(name: String) {
    println("Hello, $name!")
}

// Return Type: Function ke baad : Int lagaya
fun addNumbers(a: Int, b: Int): Int {
    return a + b
}

// Functions ko call karna
greetUser("Aman") // Output: Hello, Aman!
val sum = addNumbers(5, 10)
println(sum) // Output: 15

Single-Expression Functions

Agar aapka function sirf ek line ka hai, toh aap use aur chhota kar sakte hain.

// Poora function (Normal)
fun add(a: Int, b: Int): Int {
    return a + b
}

// Single-Expression function (Short)
fun addShort(a: Int, b: Int) = a + b

Default Arguments

Aap parameters ko ek default value de sakte hain. Isse function call karte waqt us parameter ko skip kar sakte hain.

fun greetUser(name: String = "Guest") {
    println("Hello, $name!")
}

greetUser("MSMAXPRO") // Output: Hello, MSMAXPRO!
greetUser()            // Output: Hello, Guest! (Default value use hui)

Control Flow: if, else, and when

Control flow se aapka code "decisions" le sakta hai.

`if-else` Statements

Yeh sabse basic condition hai. "Agar yeh sach hai, toh A karo, varna B karo."

val age = 19

if (age > 18) {
    println("You can vote.")
} else {
    println("You cannot vote yet.")
}

`if-else` as an Expression

Kotlin mein `if-else` ek value bhi return kar sakta hai. Isse aapka code chhota ho jaata hai.

val age = 19
val message = if (age > 18) {
    "You can vote."
} else {
    "You cannot vote yet."
}
println(message) // Output: You can vote.

`when` Statement (Kotlin's "super-switch")

`when` statement Java ke `switch` se kahin zyada powerful hai.

val day = 3

// 'when' as a statement
when (day) {
    1 -> println("Monday")
    2 -> println("Tuesday")
    3 -> println("Wednesday")
    else -> println("Other day")
}

// 'when' as an expression (aur bhi behtar)
val dayName = when (day) {
    1 -> "Monday"
    2 -> "Tuesday"
    3 -> "Wednesday"
    else -> "Other day"
}
println(dayName) // Output: Wednesday

Collections and Loops

Collections data ke groups ko store karne ke liye hote hain (jaise lists ya maps).

Lists

Lists items ka ek ordered group hota hai. Kotlin mein 2 tarah ki lists hain: read-only aur mutable.

// 1. Read-only List (Aap ise change nahi kar sakte)
val web = listOf("HTML", "CSS", "JS")
println(web[0]) // Output: HTML (index 0 se start hota hai)
// web.add("React") // ERROR!

// 2. Mutable List (Aap ise change kar sakte hain)
val mobile = mutableListOf("Android", "iOS")
mobile.add("Flutter")
println(mobile) // Output: [Android, iOS, Flutter]

`for` Loops

`for` loop ka istemaal list (ya range) ke har item par code chalaane ke liye hota hai.

val web = listOf("HTML", "CSS", "JS")

// 'tech' variable har item ko ek-ek karke represent karega
for (tech in web) {
    println("Learn $tech")
}
// Output:
// Learn HTML
// Learn CSS
// Learn JS

Introduction to Classes (OOP)

Kotlin ek Object-Oriented Programming (OOP) language hai. Classes objects banane ke liye **blueprints** hote hain. Ek "User" class ek blueprint hai, aur *aap* (MSMAXPRO) us class ke ek object (ya instance) hain.

Defining a Class

Yeh ek simple `User` class hai. `username` aur `age` iske "properties" hain. `greet` iska ek "method" (function) hai.

class User(val username: String, val age: Int) {
    
    // Yeh ek member function (method) hai
    fun greet() {
        println("Hello, my name is $username and I am $age years old.")
    }
}

Creating Objects (Instances)

Ek baar blueprint (class) taiyyar ho jaaye, toh aap usse objects bana sakte hain.

fun main() {
    // 'user1' ek User object (instance) hai
    val user1 = User("MSMAXPRO", 20)
    
    // 'user2' ek doosra User object hai
    val user2 = User("Aman", 22)

    // Properties access karna
    println(user1.username) // Output: MSMAXPRO

    // Methods call karna
    user2.greet() // Output: Hello, my name is Aman and I am 22 years old.
}

Data Classes

Kotlin mein ek shortcut hai. Agar aapki class sirf data store karne ke liye hai, toh aap `data` keyword ka use kar sakte hain. Isse Kotlin aapke liye bahut saara code (jaise `.equals()`, `.toString()`) automatically bana deta hai.

// Sirf 'data' keyword add kiya
data class Book(val title: String, val author: String)

fun main() {
    val book1 = Book("Atomic Habits", "James Clear")
    
    // .toString() automatically mil gaya
    println(book1) 
    // Output: Book(title=Atomic Habits, author=James Clear)
}
