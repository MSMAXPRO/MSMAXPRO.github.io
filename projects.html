Full Stack Projects

Apply Your Frontend & Backend Skills

You've completed the roadmaps, now it's time to connect the frontend and backend to build complete, impressive applications. These projects will challenge you to use the MERN stack (MongoDB, Express, React, Node.js) to create dynamic, data-driven websites. For each project, we'll outline the features, the tech stack, and the logic you'll need to implement.

1. Full Stack Blog Platform

A complete blog where users can register, log in, create their own posts, and view posts from others. This is a fantastic portfolio project that demonstrates all the core full-stack skills.

  • Tech Stack: React (Frontend), Express/Node.js (Backend), MongoDB (Database), JWT for authentication.
  • Skills Practiced: Full CRUD API, User Authentication, Password Hashing, Protected Routes, State Management in React, Fetching Data.

Backend Features (The API):

  1. User Model: Create a Mongoose schema for `User` with `username`, `email`, and `password`.
  2. Post Model: Create a Mongoose schema for `Post` with `title`, `content`, `author` (linking to a User ID), and `createdAt`.
  3. Auth Routes:
    • `POST /api/auth/signup`: Hashes password with `bcrypt` and creates a new user.
    • `POST /api/auth/login`: Checks credentials, compares hashed passwords, and returns a JWT on success.
  4. Post Routes (CRUD):
    • `GET /api/posts`: Fetches all blog posts.
    • `GET /api/posts/:id`: Fetches a single post by its ID.
    • `POST /api/posts`: A **protected route**. Creates a new post. The user ID from the JWT should be saved as the `author`.
    • `PATCH /api/posts/:id`: A **protected route**. Updates a post. Add logic to ensure only the original author can edit their own post.
    • `DELETE /api/posts/:id`: A **protected route**. Deletes a post. Add logic to ensure only the author can delete it.
  5. Middleware: Create an `authenticateToken` middleware to protect the POST, PATCH, and DELETE routes by verifying the JWT.

Frontend Features (The React App):

  1. Components: Create components like `Navbar`, `PostList`, `PostDetail`, `CreatePostForm`, `LoginForm`, `SignupForm`.
  2. Routing: Use a library like `React Router` to create different pages (`/`, `/posts/:id`, `/login`, `/signup`, `/create-post`).
  3. State Management: Use `useState` and `useEffect` hooks. For managing login status across the app, consider using `useContext`.
  4. API Calls: Use `fetch()` or `axios` to make requests to your backend API.
    • Fetch all posts on the homepage.
    • Send user details to `/signup` or `/login`.
    • When a user logs in, save the received JWT to `localStorage`.
    • For every request to a protected API route, include the JWT in the `Authorization` header.
  5. **Protected Routes:** In React, create a component that checks if a user is logged in (by checking for the JWT in `localStorage`). If not, redirect them to the login page.

2. Simple Task Manager (Trello Clone)

A web application where authenticated users can manage their tasks in different columns (e.g., "To Do", "In Progress", "Done").

  • Tech Stack: React (with a drag-and-drop library like `react-beautiful-dnd`), Express, MongoDB, JWT.
  • Skills Practiced: Data Modeling (relationships), Complex CRUD, User Authentication, advanced frontend state management.

Backend Features:

  1. User Model: Standard user schema with `email`, `password`.
  2. Task Model: A Mongoose schema for `Task` with `title`, `description`, `status` (e.g., 'todo', 'inprogress', 'done'), and `owner` (linking to the User ID).
  3. Auth Routes: Standard `/signup` and `/login` routes returning a JWT.
  4. Task Routes (Protected):
    • `GET /api/tasks`: Fetches all tasks belonging *only* to the logged-in user (use the ID from the JWT).
    • `POST /api/tasks`: Creates a new task for the logged-in user.
    • `PATCH /api/tasks/:id`: Updates a task (e.g., changing its `status` when moved to a new column, or editing its text).
    • `DELETE /api/tasks/:id`: Deletes a task.

Frontend Features:

  1. UI/Layout: Create a three-column layout for "To Do", "In Progress", and "Done".
  2. State Management: The frontend state will be an object or map where keys are the column names and values are arrays of task objects.
  3. Drag and Drop: Implement a library like `react-beautiful-dnd` to allow users to drag task cards between columns.
  4. API Calls:
    • On page load, `fetch` all tasks for the logged-in user and organize them into the correct columns in your state.
    • When a task is dropped into a new column, update the frontend state immediately for a smooth UX, then send a `PATCH` request to your backend to update the task's `status` in the database.
    • When creating or deleting a task, send the appropriate request and then update the local state to reflect the change.

3. URL Shortener Full Stack

A full application based on the backend-only project. Users can (optionally) log in to see a history of all the URLs they have shortened.

  • Tech Stack: React, Express, MongoDB. (Authentication is optional but a great addition).
  • Skills Practiced: API design, frontend-backend integration, handling redirects.

Backend Features:

  1. URL Model: Schema with `originalUrl`, `shortCode`, and an optional `creator` field (if you add user accounts).
  2. API Routes:
    • `POST /shorten`: Takes `originalUrl`, generates a `shortCode`, saves it, and returns the shortened link.
    • `GET /:shortCode`: Finds the code in the database and `res.redirect()`s to the `originalUrl`.
    • (Optional) `GET /api/my-urls`: A protected route that returns all URLs created by the logged-in user.

Frontend Features:

  1. UI: A simple input field for the long URL and a button.
  2. API Calls:
    • When the user submits a URL, send a `POST` request to your `/shorten` API.
    • Display the shortened URL returned by the backend.
    • Add a "copy to clipboard" button for user convenience.
  3. (Optional) Add Login/Signup pages and a "My URLs" page that fetches and displays the user's link history.
