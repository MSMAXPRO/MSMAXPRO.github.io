Project Idea: Weather Dashboard

Fetch and display live data from a third-party API.

This is a perfect project to understand how to work with Third-Party APIs. We will build a simple dashboard where a user can search for any city name, and our app will call an external API (like OpenWeatherMap) to fetch and display the current temperature, humidity, and weather conditions. With this project, you will master asynchronous JavaScript and handling API data.

Frontend Features (The React App)

  1. Get API Key: First, go to OpenWeatherMap, create a free account, and get your unique API Key.
  2. Component UI: Create a `Weather` component that includes an `input` field for the city name and a `button` to trigger the search.
  3. State Management: Use `useState` hooks to manage several pieces of data:
    • `city`: To store the value from the input field.
    • `weatherData`: To store the JSON response from the API.
    • `loading`: A boolean to show a "Loading..." message.
    • `error`: To store any error messages (e.g., "City not found").
  4. API Fetching Logic:
    • Create an `async function` called `fetchWeather`.
    • Inside, wrap your logic in a `try...catch...finally` block.
    • `try`: Set `setLoading(true)`. Use `await fetch(...)` to call the OpenWeatherMap URL (include your API key and the `city` state). Check if `response.ok` is true. If not, throw an error. Convert the response with `await response.json()` and save it using `setWeatherData`.
    • `catch`: Set the `setError(error.message)`.
    • `finally`: Set `setLoading(false)`.
  5. Conditional Rendering:
    • If `loading` is true, display a loading spinner or text.
    • If `error` is not null, display the error message.
    • If `weatherData` is not null, pass it as props to a `WeatherCard` component to display the temperature, icon, etc.

After completing the project, try these ideas:

  • **5-Day Forecast:** Along with the current weather, display the forecast for the next 5 days (OpenWeatherMap provides this data too).
  • **Geolocation:** Show the weather for the user's current location (using the browser's Geolocation API) on initial page load.
  • **Save Last Search:** Save the user's last searched city in `localStorage` so it becomes the default location when the app opens.
