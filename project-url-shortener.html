Project Idea: URL Shortener

Build a full-stack MERN tool like Bit.ly.

This is a classic full-stack utility project. You will build a tool that takes a very long URL from a user and gives them back a short, unique link (e.g., `mysite.com/a8f3H`). When anyone visits that short link, your app will automatically redirect them to the original long URL. This project is fantastic for understanding the full MERN stack flow.

  • An input field to paste a long URL.
  • A "Shorten" button that generates a unique short code.
  • Display the new, short link to the user.
  • When the short link is visited, it must redirect to the original long URL.

Backend Features (The API)

  1. URL Model: Create a Mongoose schema for `Url` with fields like `longUrl` (String), `shortCode` (String, unique), `clicks` (Number, default 0), and `createdAt`.
  2. Shorten Route (`POST /api/shorten`):
    • Receives the `longUrl` from the React app.
    • Validates if it's a real URL.
    • Checks if this `longUrl` already exists in the database. If yes, return the existing short link.
    • If not, generate a new unique `shortCode` using a library like `nanoid`.
    • Save the new `Url` object (with `longUrl` and `shortCode`) to MongoDB.
    • Send back the full short URL (e.g., `http://localhost:5000/a8f3H`) to the React app.
  3. Redirect Route (`GET /:shortCode`):
    • This is the most important route. It's a top-level route, NOT under `/api`.
    • It captures the short code from the URL (e.g., `a8f3H`).
    • Finds the document in MongoDB with that `shortCode`.
    • If not found, send a 404 error.
    • If found, increment the `clicks` count.
    • Finally, use `res.redirect(url.longUrl)` to send the user to the original long URL.

Frontend Features (The React App)

  1. Components: A simple UI with a `header`, a `URLInputForm` component, and a `ShortLinkDisplay` component.
  2. State Management: Use `useState` to manage the long URL input (`longUrl`), the returned short link (`shortUrl`), loading, and error states.
  3. Form Handling:
    • Create a form with an `input` and a `button`.
    • On submit, prevent the default form action.
    • Set `loading` to true.
  4. API Call:
    • Make a `POST` request to your backend (`/api/shorten`) using `fetch` or `axios`.
    • Send the `longUrl` in the request body.
    • When the response comes back, save the new short link to the `shortUrl` state.
    • Display the `shortUrl` state on the screen for the user to copy.
