Project Idea: Full Stack Blog

Build a complete MERN stack blogging platform.

A complete blog where users can register, log in, create their own posts, and view posts from others. This is a fantastic portfolio project that demonstrates all the core full-stack skills: authentication, database relationships, and full CRUD operations.

âœ¨ Key Features

  • Secure user registration and login with JWT (JSON Web Tokens).
  • Full Create, Read, Update, Delete (CRUD) functionality for blog posts.
  • Protected routes: Only logged-in users can create posts.
  • Authorization: Only the author of a post can edit or delete it.
  • A public homepage to read all posts.

Backend Features (The API)

  1. User Model: Create a Mongoose schema for `User` with `username`, `email`, and `password`.
  2. Post Model: Create a Mongoose schema for `Post` with `title`, `content`, `author` (linking to a User ID using `mongoose.Schema.Types.ObjectId`), and `createdAt`.
  3. Auth Routes:
    • `POST /api/auth/signup`: Hashes password with `bcrypt` and creates a new user.
    • `POST /api/auth/login`: Checks credentials, compares hashed passwords, and returns a JWT on success.
  4. Post Routes (CRUD):
    • `GET /api/posts`: Fetches all blog posts (and populate author info).
    • `GET /api/posts/:id`: Fetches a single post by its ID.
    • `POST /api/posts`: A protected route. Creates a new post. The user ID from the JWT should be saved as the `author`.
    • `PATCH /api/posts/:id`: A protected route. Updates a post. Add logic to ensure only the original author can edit their own post.
    • `DELETE /api/posts/:id`: A protected route. Deletes a post. Add logic to ensure only the author can delete it.
  5. Middleware: Create an `authenticateToken` middleware to protect the POST, PATCH, and DELETE routes by verifying the JWT.

Frontend Features (The React App)

  1. Components: Create components like `Navbar`, `PostList`, `PostDetail`, `CreatePostForm`, `LoginForm`, `SignupForm`.
  2. Routing: Use a library like `React Router` to create different pages (`/`, `/posts/:id`, `/login`, `/signup`, `/create-post`).
  3. State Management: Use `useState` and `useEffect` hooks. For managing login status across the app, consider using `useContext` to create an `AuthContext`.
  4. API Calls: Use `fetch()` or `axios` to make requests to your backend API.
    • Fetch all posts on the homepage.
    • Send user details to `/signup` or `/login`.
    • When a user logs in, save the received JWT to `localStorage` and update the `AuthContext`.
    • For every request to a protected API route, include the JWT in the `Authorization` header (e.g., `Bearer ${token}`).
