Project Idea: E-commerce Cart

Master global state in React with a functional cart.

This project is essential for every React developer's portfolio. We will build a professional-looking product page. When a user clicks 'Add to Cart', the item will be added to a global cart state. The user can then navigate to the `/cart` page to see all their items. The main focus is Global State Management using the Context API to avoid "prop drilling".

  • A product page showing item details.
  • "Add to Cart" button to add items to a global cart.
  • A separate "Cart" page (`/cart`) to display all selected items.
  • Ability to see the total number of items in the cart (e.g., in the navbar).
  • A global state that is accessible from any component.

Frontend Features (The React App)

  1. Routing Setup: Use `react-router-dom` to create two main routes: `/` (ProductsPage) and `/cart` (CartPage). Create a `Navbar` component that links to both.
  2. Context API Setup:
    • Create a `CartContext.js` file.
    • Use `createContext()` to initialize a new context.
    • Create a `CartProvider` component that will wrap your entire application.
    • Inside the provider, use `useState` or `useReducer` to manage the `cartItems` state.
    • Create functions like `addToCart`, `removeFromCart` and pass them in the context `value`.
  3. Products Page Component:
    • Create a list of dummy product data (as an array of objects).
    • Map over this array to display each product.
    • Import the `useContext(CartContext)` hook and call the `addToCart` function on button click.
  4. Cart Page Component:
    • Use `useContext(CartContext)` to access the `cartItems` state array.
    • Map over the array to display the items in the cart.
    • Add functionality to call `removeFromCart`.
  5. Navbar Component: Use `useContext` again to access `cartItems.length` and display the number of items in the cart badge.

ðŸ”¥ Next Steps & Challenges

After completing the project, try these ideas:

  • Use `useReducer`: For more complex state, convert your `useState` logic in the context to a `useReducer` hook.
  • **Quantity:** Add a feature to increase or decrease item quantity on the cart page.
  • **Local Storage:** Save the cart items to `localStorage` so the cart isn't empty after a page reload.
  • **Backend:** Fetch products from a fake API (like `fakestoreapi.com`) instead of using dummy data.
